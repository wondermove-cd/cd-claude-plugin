# Manual Template Skill

## 목적

일관된 구조와 스타일의 사용자 매뉴얼을 작성하도록 돕습니다.

## 활성화 조건

다음 상황에서 자동 활성화:
- `/ux manual` 커맨드 실행 시
- "매뉴얼", "사용자 가이드", "PPT" 키워드 언급 시
- sections/ 폴더 내 파일 작성 시

## 매뉴얼 작성 원칙

### 1. 사용자 중심

**원칙**: 사용자의 목표와 작업 흐름 중심으로 구성

❌ 나쁜 예 (기능 중심):
```
3. API 설정
4. 데이터베이스 연결
5. 서버 배포
```

✅ 좋은 예 (작업 중심):
```
3. 시작하기
   3-1. 프로젝트 생성하기
   3-2. 첫 번째 앱 만들기
   3-3. 배포하기
```

### 2. 단계별 구성

**구조**:
```
1. 목표 제시 (무엇을 할 수 있는가)
2. 사전 준비 (필요한 것)
3. 단계별 실행 (어떻게 하는가)
4. 결과 확인 (성공 여부 판단)
```

### 3. 시각 자료 우선

**원칙**: 텍스트보다 스크린샷/다이어그램 우선

**비율**:
- 기능 소개 페이지: 40% 텍스트 + 60% 이미지
- 단계별 가이드: 30% 텍스트 + 70% 이미지
- 표/비교: 70% 텍스트 + 30% 이미지

## 페이지 템플릿

### 템플릿 1: 기능 소개 페이지

**사용 시기**: 새로운 기능을 처음 소개할 때

**구조**:

```markdown
# {기능명}

> {한 줄 설명 - 이 기능으로 무엇을 할 수 있는가}

## 기능 설명

{2-3 문단}
- 첫 문단: 이 기능이 무엇인지
- 둘째 문단: 왜 유용한지
- 셋째 문단: 언제 사용하는지

이 기능을 통해 사용자는...

## 주요 특징

📊 **특징 1** - {한 줄 설명}
⚡ **특징 2** - {한 줄 설명}
🎯 **특징 3** - {한 줄 설명}

## 화면 구성

[스크린샷: {파일명}.png]

### 주요 영역 설명

1. **{영역 1 이름}** - {역할 설명}
2. **{영역 2 이름}** - {역할 설명}
3. **{영역 3 이름}** - {역할 설명}

---

*다음: [{다음 섹션}]({링크})*
```

**PPT 레이아웃**:
```
+------------------+------------------+
| 제목             |                  |
| 한 줄 설명       |                  |
|                  |   스크린샷       |
| 특징 1          |   (우측 60%)     |
| 특징 2          |                  |
| 특징 3          |                  |
+------------------+------------------+
```

### 템플릿 2: 단계별 가이드 페이지

**사용 시기**: 작업 수행 방법을 설명할 때

**구조**:

```markdown

> {이 작업으로 달성할 수 있는 것}

## 사전 준비

이 작업을 수행하기 전에 다음을 확인하세요:
- [ ] {전제 조건 1}
- [ ] {전제 조건 2}

---

## Step 1: {첫 번째 단계}

{동작 설명 - "클릭합니다", "입력합니다" 형태}

[스크린샷: {파일명}_step1.png]

💡 **Tip**: {추가 도움말}

---

## Step 2: {두 번째 단계}

{동작 설명}

[스크린샷: {파일명}_step2.png]

⚠️ **주의**: {주의사항}

---

## Step 3: {세 번째 단계}

{동작 설명}

[스크린샷: {파일명}_step3.png]

## 결과 확인

작업이 성공적으로 완료되면:
- ✅ {결과 1}
- ✅ {결과 2}

❌ **문제 해결**
- 만약 {문제 상황}이 발생하면: {해결 방법}

```

```
| 💡 Tip: {도움말}                |

```

## 스타일 가이드

### 제목 계층

```markdown
# H1: 페이지 제목 (슬라이드당 1개)
## H2: 주요 섹션
### H3: 세부 항목
```

### 강조

```markdown
**굵게**: 중요 키워드
*기울임*: 변수명, 파일명
`코드`: 입력값, 설정명
```

### 아이콘 사용

| 용도 | 아이콘 |
|------|--------|
| 팁 | 💡 |
| 주의 | ⚠️ |
| 성공 | ✅ |
| 오류 | ❌ |
| 정보 | ℹ️ |
| 설정 | ⚙️ |
| 검색 | 🔍 |

### 목록

**순서 있는 목록** (단계):
```markdown
1. 첫 번째 단계
2. 두 번째 단계
3. 세 번째 단계
```

**순서 없는 목록** (항목):
```markdown
- 항목 1
- 항목 2
- 항목 3
```

**체크리스트**:
```markdown
- [ ] 미완료 항목
- [x] 완료 항목
```

## 스크린샷 가이드

### 파일명 규칙

```
{섹션번호}_{기능명}_{유형}_{순서}.png

예시:
03_02_monitoring_main.png        (메인 화면)
03_02_monitoring_step1.png       (1단계)
03_02_monitoring_comparison.png  (비교 화면)
```

### 하이라이트 규칙

1. **빨간 테두리** (2px): 주요 버튼, 입력 필드
2. **화살표**: 클릭/입력해야 할 위치
3. **번호 (1, 2, 3)**: 순서가 있는 단계
4. **모자이크**: 개인정보, API 키 등 민감 정보

### 이미지 최적화

- **해상도**: 1920x1080 (Full HD)
- **포맷**: PNG (투명 배경 지원)
- **파일 크기**: 2MB 이하 (품질 80-90%)
- **압축**: TinyPNG 등 도구 사용

## 작성 체크리스트

매뉴얼 페이지 작성 완료 전:

- [ ] **구조**
  - 템플릿 구조 준수
  - 제목 계층 올바름
  - 섹션 순서 논리적

- [ ] **내용**
  - 사용자 작업 중심 설명
  - 전문 용어 최소화 (또는 설명 추가)
  - 동사로 시작하는 명령문

- [ ] **스크린샷**
  - 모든 [스크린샷: ...] 태그에 실제 파일 존재
  - 파일명 규칙 준수
  - 하이라이트 적절히 표시
  - 민감정보 모자이크 처리

- [ ] **스타일**
  - 존댓말 사용
  - 아이콘 일관성
  - 목록 형식 통일

- [ ] **링크**
  - 관련 페이지 링크 정확
  - 외부 링크 유효성 확인

```
   - 03_02_monitoring_step3.png
   - 03_02_monitoring_result.png

   - screenshot1.png → 03_02_monitoring_main.png

개선 제안:
1. assets/screenshots/ 폴더에 누락된 스크린샷 추가
2. 파일명을 규칙에 맞게 변경
```
