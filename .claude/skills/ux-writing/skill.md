# UX Writing Skill

## 목적

사용자 경험을 고려한 명확하고 일관된 문구를 작성하도록 돕습니다.

## 활성화 조건

다음 상황에서 자동 활성화:
- 기획서 작성 시
- 버튼명, 메뉴명 정의 시
- 오류 메시지, 안내 문구 작성 시
- "문구", "텍스트", "메시지" 키워드 언급 시

## UX 라이팅 원칙

### 1. 명확성 (Clarity)

**원칙**: 사용자가 한 번에 이해할 수 있어야 합니다.

❌ 나쁜 예:
```
처리가 불가능합니다.
```

✅ 좋은 예:
```
파일 크기가 10MB를 초과하여 업로드할 수 없습니다.
```

### 2. 간결성 (Conciseness)

**원칙**: 필요한 정보만 전달합니다.

❌ 나쁜 예:
```
저장 버튼을 클릭하시면 입력하신 내용이 서버에 저장됩니다.
```

✅ 좋은 예:
```
저장
```

### 3. 일관성 (Consistency)

**원칙**: 같은 의미는 같은 표현으로 통일합니다.

| 기능 | 사용 | 사용 금지 |
|------|------|----------|
| 삭제 | "삭제" | "제거", "지우기" |
| 수정 | "수정" | "편집", "변경" |
| 확인 | "확인" | "OK", "알겠습니다" |

### 4. 존중 (Tone)

**원칙**: 사용자를 존중하는 톤을 유지합니다.

❌ 나쁜 예:
```
잘못된 입력입니다.
```

✅ 좋은 예:
```
올바른 이메일 형식을 입력해주세요.
```

## 문구 유형별 가이드

### 버튼 (Buttons)

**규칙**:
- 동사로 시작
- 2-3 단어 이내
- 결과를 명확히 표현

| 상황 | 버튼명 |
|------|--------|
| 데이터 저장 | "저장" |
| 설정 적용 | "적용" |
| 작업 취소 | "취소" |
| 삭제 확인 | "삭제" (위험한 동작은 빨간색) |
| 다음 단계 | "다음" |
| 완료 | "완료" |

**긴급/위험 동작**:
```
"삭제" → "영구 삭제"
"초기화" → "모두 초기화"
```

### 오류 메시지 (Error Messages)

**구조**:
```
[문제] + [원인] + [해결 방법]
```

**예시**:

❌ 나쁜 예:
```
오류가 발생했습니다.
```

✅ 좋은 예:
```
파일을 업로드할 수 없습니다.
파일 크기가 10MB를 초과했습니다.
10MB 이하의 파일을 선택해주세요.
```

**템플릿**:
```
{동작}할 수 없습니다.
{원인}
{해결 방법을 동사로 시작}해주세요.
```

### 성공 메시지 (Success Messages)

**규칙**:
- 간결하게
- 완료된 동작을 명시
- 다음 단계 안내 (필요시)

**예시**:

```
✅ 저장되었습니다.
✅ 프로젝트가 생성되었습니다.
✅ 초대 이메일을 발송했습니다.
```

### 안내 문구 (Helper Text)

**규칙**:
- 입력 필드 아래 배치
- 회색 텍스트
- "~합니다", "~할 수 있습니다" 형태

**예시**:

```
비밀번호는 8자 이상이어야 합니다.
프로필 사진은 언제든지 변경할 수 있습니다.
```

### 빈 상태 (Empty State)

**구조**:
```
[제목: 현재 상태 설명]
[부제: 행동 유도]
[버튼: 다음 동작]
```

**예시**:

```
아직 프로젝트가 없습니다.
첫 번째 프로젝트를 생성해보세요.

[+ 새 프로젝트 만들기]
```

## 톤 앤 매너

### 기본 원칙

| 원칙 | 설명 | 예시 |
|------|------|------|
| **존댓말** | 항상 "~합니다", "~주세요" | "저장되었습니다" (O) |
| **능동형** | 능동적 표현 선호 | "확인할 수 있습니다" (O), "확인됩니다" (X) |
| **긍정형** | 부정보다 긍정 | "이메일을 입력해주세요" (O), "이메일이 없습니다" (X) |

### 상황별 톤

**일반 상황**:
```
친절하고 전문적
예) "프로젝트를 생성하시겠습니까?"
```

**오류 상황**:
```
공감하고 해결 중심
예) "일시적인 문제가 발생했습니다. 잠시 후 다시 시도해주세요."
```

**성공 상황**:
```
간결하고 긍정적
예) "저장되었습니다."
```

## 실전 체크리스트

기획서 작성 시 다음을 자동 검증:

- [ ] 버튼명: 동사로 시작, 2-3 단어
- [ ] 오류 메시지: 문제 + 원인 + 해결 방법 포함
- [ ] 일관성: 같은 동작은 같은 표현
- [ ] 존댓말: 모든 문구에 적용
- [ ] 명확성: 전문 용어 최소화, 필요시 설명 추가

## 위반 발견 시 자동 제안

```
[UX WRITING] 개선 제안

❌ 현재: "처리 실패"
✅ 제안: "파일을 업로드할 수 없습니다. 파일 형식을 확인해주세요."

이유: 오류 메시지는 [문제 + 원인 + 해결 방법] 구조 필요
```
