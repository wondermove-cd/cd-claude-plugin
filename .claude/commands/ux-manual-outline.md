# UX Manual Outline - 매뉴얼 목차 생성

## 목적

`FUNCTIONAL_REQUIREMENTS.md`를 분석하여 매뉴얼의 목차 구조를 자동으로 생성합니다.

## 실행 절차

### Step 1: 매뉴얼 프로젝트 확인

`.ux-docs/manuals/` 디렉토리에서 매뉴얼 프로젝트 찾기:

- 프로젝트가 여러 개인 경우: 사용자에게 선택 요청
- 프로젝트가 없는 경우: `/ux manual init` 실행 제안

### Step 2: 기능 요건 분석

`.ux-docs/FUNCTIONAL_REQUIREMENTS.md` 파일 읽기:

```markdown
## 기능 카테고리 추출
1. 대분류 (예: 클러스터 관리, 모니터링, 설정)
2. 중분류 (각 대분류의 세부 기능)
3. 소분류 (각 기능의 세부 동작)
```

### Step 3: 목차 구조 설계

#### 표준 매뉴얼 구조

```
00. 표지
01. 서비스 개요
02. 시작하기
03. 주요 기능
    03-01. {대분류 1}
        03-01-01. {기능 1}
        03-01-02. {기능 2}
    03-02. {대분류 2}
        03-02-01. {기능 3}
        03-02-02. {기능 4}
04. 설정 및 관리
05. 문제 해결 (FAQ)
06. 부록
```

### Step 4: MANUAL_OUTLINE.md 생성

```markdown
# {제품명} 매뉴얼 목차

## 문서 정보

| 항목 | 내용 |
|------|------|
| 제품명 | {제품명} |
| 버전 | v1.0 |
| 총 페이지 | {계산된 페이지 수} |

---

## 목차 구조

### 00. 표지
- 파일: sections/00_cover.md
- 페이지 수: 1

### 01. 서비스 개요 (2-3페이지)
- 파일: sections/01_overview.md
- 내용:
  - 01-01. {제품명}란?
  - 01-02. 주요 기능
  - 01-03. 시스템 요구사항

### 02. 시작하기 (3-5페이지)
- 파일: sections/02_getting_started.md
- 내용:
  - 02-01. 접속 방법
  - 02-02. 로그인
  - 02-03. 초기 설정
  - 02-04. 화면 구성 소개

### 03. 주요 기능 ({n}페이지)

#### 03-01. {대분류 1} ({n}페이지)
- 파일: sections/03_01_{대분류}.md

**03-01-01. {기능 1}**
- 페이지 유형: 기능 소개
- 스크린샷: 03_01_01_{기능}.png

**03-01-02. {기능 2}**
- 페이지 유형: 단계별 가이드
- 스크린샷:
  - 03_01_02_{기능}_step1.png
  - 03_01_02_{기능}_step2.png

#### 03-02. {대분류 2} ({n}페이지)
- 파일: sections/03_02_{대분류}.md

...

### 04. 설정 및 관리 (2-3페이지)
- 파일: sections/04_settings.md
- 내용:
  - 04-01. 사용자 설정
  - 04-02. 권한 관리
  - 04-03. 알림 설정

### 05. 문제 해결 (FAQ) (2-3페이지)
- 파일: sections/05_faq.md
- 내용:
  - 자주 묻는 질문 10-15개

### 06. 부록 (1-2페이지)
- 파일: sections/06_appendix.md
- 내용:
  - 용어 사전
  - 단축키 목록
  - 고객 지원 안내

*목차는 /ux manual page 실행 시마다 자동으로 업데이트됩니다.*
```

### Step 5: 섹션 파일 스켈레톤 생성

각 섹션별로 빈 마크다운 파일 생성:

```markdown
# {섹션 제목}

{MANUAL_STYLE_GUIDE.md의 해당 페이지 유형 템플릿}

## TODO
- [ ] 내용 작성
- [ ] 스크린샷 준비
- [ ] 리뷰 완료
```

### Step 6: 완료 보고

```
============================================
 [UX MANUAL OUTLINE] 목차 생성 완료
============================================

 매뉴얼: {제품명}

 생성된 구조:
 ✅ MANUAL_OUTLINE.md (목차)
 ✅ {n}개 섹션 파일 (스켈레톤)

 예상 페이지 수: {n}페이지

 작성 우선순위:
 📌 P0 (필수): {n}개 섹션
 📋 P1 (중요): {n}개 섹션
 📄 P2 (부가): {n}개 섹션

 필요한 스크린샷: {n}개

 다음 단계:
 1. 우선순위 높은 섹션부터 작성:
    /ux manual page "서비스 개요"
 2. 스크린샷 준비:
    assets/screenshots/ 폴더에 저장

 💡 Tip:
 - MANUAL_OUTLINE.md에서 전체 구조 확인
 - 각 섹션 파일은 TODO 체크리스트 포함

============================================
```

## 참조 파일

- `.ux-docs/FUNCTIONAL_REQUIREMENTS.md` - 기능 목록
- `.ux-docs/manuals/{제품명}/MANUAL_STYLE_GUIDE.md` - 스타일 가이드
