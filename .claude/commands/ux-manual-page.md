# UX Manual Page - 매뉴얼 페이지 작성

## 목적

MANUAL_OUTLINE.md의 특정 섹션에 대한 상세 내용을 작성합니다.

## 실행 절차

### Step 1: 섹션명 확인

$ARGUMENTS에서 섹션명 파악 또는 MANUAL_OUTLINE.md 기반 선택지 제시:

```
어떤 섹션을 작성하시겠습니까?

1. 서비스 개요
2. 시작하기
3. 클러스터 관리
4. 모니터링
...

또는 직접 입력: _____
```

### Step 2: 섹션 정보 로드

MANUAL_OUTLINE.md에서 해당 섹션 정보 추출:

- 파일 경로
- 페이지 유형
- 필요한 스크린샷 목록

### Step 3: 관련 컨텍스트 로드

```
1. FUNCTIONAL_REQUIREMENTS.md - 기능 요구사항
2. UX_PATTERNS.md - UX 패턴
3. MANUAL_STYLE_GUIDE.md - 작성 스타일
```

### Step 4: 페이지 유형별 작성

#### 유형 1: 기능 소개 페이지

```markdown
# {기능명}

> {한 줄 설명}

{FUNCTIONAL_REQUIREMENTS.md 기반으로 2-3 문단 작성}

## 주요 특징

📊 **특징 1** - {설명}
⚡ **특징 2** - {설명}
🎯 **특징 3** - {설명}

## 화면 구성

[스크린샷: {섹션}_{기능}_main.png]

### 주요 영역 설명

1. **영역 1** - {설명}
2. **영역 2** - {설명}
3. **영역 3** - {설명}

---

## TODO
- [ ] 스크린샷 촬영: {섹션}_{기능}_main.png
- [ ] 내용 검토
- [ ] 용어 통일 확인
```

## 스크린샷 촬영 가이드

### 촬영 팁

1. **전체 화면 vs 부분 캡처**
   - 전체 화면: 레이아웃 소개, 메뉴 구조
   - 부분 캡처: 특정 기능, 버튼, 입력 필드

2. **하이라이트 표시**
   - 빨간 테두리: 주요 버튼, 입력 필드
   - 화살표: 클릭/입력해야 할 위치
   - 번호: 순서가 있는 단계

3. **품질 기준**
   - 해상도: 최소 1920x1080
   - 포맷: PNG (배경 투명 가능)
   - 파일 크기: 2MB 이하 권장

## 참조 파일

- `.ux-docs/manuals/{제품명}/MANUAL_OUTLINE.md` - 전체 목차
- `.ux-docs/manuals/{제품명}/MANUAL_STYLE_GUIDE.md` - 작성 규칙
- `.ux-docs/FUNCTIONAL_REQUIREMENTS.md` - 기능 요구사항
