# UX Manual Init - 매뉴얼 프로젝트 초기화

## 목적

사용자 매뉴얼 제작을 위한 프로젝트 구조를 생성하고 기본 템플릿을 준비합니다.

## 실행 절차

### Step 1: 제품명 확인

$ARGUMENTS에서 제품명 파악 또는 질문:

```
매뉴얼을 제작할 제품명을 입력해주세요:
예) SKuber v2.0, 디지털 플랫폼 관리자 가이드
```

### Step 2: 매뉴얼 디렉토리 구조 생성

```
.ux-docs/manuals/{제품명}/
├── MANUAL_OUTLINE.md           # 매뉴얼 목차 및 구조
├── MANUAL_STYLE_GUIDE.md       # 매뉴얼 작성 스타일 가이드
├── sections/                   # 각 섹션별 마크다운 파일
│   ├── 00_cover.md
│   ├── 01_overview.md
│   ├── 02_getting_started.md
│   └── .gitkeep
├── assets/
│   ├── screenshots/            # UI 스크린샷
│   ├── diagrams/               # 플로우 다이어그램
│   └── icons/                  # 아이콘, 로고
└── output/
    └── .gitkeep                # PPT 파일이 생성될 위치
```

### Step 3: MANUAL_STYLE_GUIDE.md 생성

```markdown
# {제품명} 매뉴얼 스타일 가이드

## 문서 정보

| 항목 | 내용 |
|------|------|
| 제품명 | {제품명} |
| 버전 | v1.0 |
| 작성일 | {날짜} |
| 담당자 | {작성자} |

---

## 페이지 유형

### 1. 기능 소개 페이지

**레이아웃**:
- 좌측 60%: 기능 설명
- 우측 40%: UI 스크린샷
- 하단: 주요 포인트 3개

**작성 규칙**:
- 제목: "기능명" (예: "클러스터 모니터링")
- 부제: 한 줄 설명
- 본문: 2-3 문단 (각 3-4줄)
- 포인트: 핵심 가치 3개 (아이콘 + 1줄)

**예시**:
```
# 클러스터 모니터링

> 실시간으로 클러스터 상태를 확인하고 리소스를 관리합니다.

## 기능 설명
클러스터 모니터링 대시보드를 통해...

📊 **실시간 메트릭 조회** - CPU, 메모리 사용률을 실시간으로 확인
⚠️ **알림 설정** - 임계값 초과 시 자동 알림
📈 **히스토리 분석** - 최대 30일 성능 데이터 분석
```

### 2. 단계별 가이드 페이지

**레이아웃**:
- Step 1, 2, 3 세로 구성
- 각 Step마다 번호 + 설명 + 스크린샷

**작성 규칙**:
- 제목: "~하는 방법" (예: "새 프로젝트 생성하는 방법")
- 각 단계: 동사로 시작 (예: "클릭합니다", "입력합니다")
- 스크린샷: 해당 UI 영역 하이라이트

**예시**:
```
# 새 프로젝트 생성하는 방법

## Step 1: 프로젝트 메뉴 진입
좌측 사이드바에서 **프로젝트** 메뉴를 클릭합니다.
[스크린샷: sidebar_project.png]

## Step 2: 생성 버튼 클릭
우측 상단의 **+ 새 프로젝트** 버튼을 클릭합니다.
[스크린샷: create_button.png]

## Step 3: 정보 입력
프로젝트명과 설명을 입력하고 **생성** 버튼을 클릭합니다.
[스크린샷: create_form.png]
```

### 3. 표/비교 페이지

**레이아웃**:
- 제목 + 설명
- 비교표 (3-5열)
- 하단 참고사항

**작성 규칙**:
- 테이블 형식
- 명확한 컬럼명
- 각 셀: 1-2줄

---

## 스크린샷 규칙

### 파일명 규칙

```
{섹션번호}_{기능명}_{순서}.png

예시:
03_02_cluster_monitoring_01.png
03_02_cluster_monitoring_02.png
```

### 스크린샷 가이드라인

1. **해상도**: 최소 1920x1080
2. **포맷**: PNG (투명 배경 가능)
3. **하이라이트**: 중요 영역은 빨간 테두리 또는 화살표 표시
4. **민감정보**: 개인정보, API 키 등은 모자이크 처리

### 삽입 방법

마크다운에서:
```markdown
[스크린샷: 파일명.png]
```

최종 PPT 빌드 시 자동으로 이미지 삽입

---

## 용어 사용 규칙

| 용어 | 사용 | 사용 금지 |
|------|------|----------|
| 버튼 | "클릭합니다" | "누릅니다", "선택합니다" |
| 입력 | "입력합니다" | "쓰세요", "기입하세요" |
| 메뉴 | "메뉴를 클릭합니다" | "메뉴로 갑니다" |

---

## 톤 앤 매너

- **존댓말 사용**: "~합니다", "~됩니다"
- **능동형 표현**: "확인할 수 있습니다" (O), "확인됩니다" (X)
- **간결성**: 한 문장 20자 이내 권장
```
