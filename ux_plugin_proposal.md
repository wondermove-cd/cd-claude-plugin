# 원더 무브 연구소
# Claude Code UX/UI 업무 동화 플러그인

> 어떤 상황에서든 동일한 UX 설계 품질을 보장하는 AI 기획 도구

---

## 1. 배경: AI 기획의 보이지 않는 비용

### 문제 상황

| 상황 | 문제점 |
|------|--------|
| **새 프로젝트** | 설계 없이 즉흥적으로 기획하여 UX 일관성이 무시됨 |
| **기존 프로젝트** | 기존 패턴을 무시하여 디자인 시스템과 충돌 |
| **작업 재개** | 이전 맥락을 잊어버려 핵심 규칙을 위반 |

**결과**: 이 모든 것이 일관성 없는 기획과 누적되는 **디자인 부채(Design Debt)**로 이어집니다.

---

## 2. 해결책: 일관성을 '강제'하는 지능형 기획 시스템

### Claude Code UX Plugin의 4가지 핵심 기능

| 기능 | 설명 |
|------|------|
| **워크플로우 강제** | 아이디어부터 핸드오프까지 체계적인 프로세스를 따름 |
| **컨텍스트 자동 학습** | 기존 프로젝트의 UX 패턴과 규칙을 스스로 파악 |
| **품질 규칙 자동 적용** | 기획자가 신경 쓰지 않아도 품질 기준을 준수 |
| **진행 상황 투명화** | 모든 작업이 실시간으로 추적되고 공유됨 |

---

## 3. 언제, 누가, 어떻게 시작하든 동일한 결과물

### 3가지 진입점

| 명령어 | 상황 | 설명 |
|--------|------|------|
| `/ux init` | 새 프로젝트 | 체계적인 기획과 설계를 통해 첫 단추를 올바르게 끼움 |
| `/ux onboard` | 기존 프로젝트 합류 | 단 몇 분 만에 프로젝트 구조와 패턴을 학습하여 즉시 기여 |
| `/ux restore-context` | 중단된 작업 재개 | 중단 지점의 모든 맥락을 즉시 복원하여 재작업 방지 |

## 4. UX 문서 기본 세트

### `/ux onboard` 또는 `/ux init` 실행 시 자동 생성

명령어 하나로 프로젝트를 분석하여 **8개의 핵심 컨텍스트 문서**를 자동 생성합니다.

```
.ux-docs/
├── PROJECT_CONTEXT.md          # 프로젝트 배경, 목표, 타겟 사용자
├── USER_RESEARCH.md            # 페르소나, 사용자 인터뷰, 페인포인트
├── INFORMATION_ARCHITECTURE.md # IA, 사이트맵, 네비게이션 구조
├── USER_FLOWS.md               # 주요 사용자 플로우 다이어그램
├── FUNCTIONAL_REQUIREMENTS.md  # 기능 요건 정의서
├── UX_PATTERNS.md              # 프로젝트에서 사용하는 UX 패턴
├── DESIGN_TOKENS.md            # Figma에서 추출한 디자인 토큰 정리
├── SPEC_CONVENTIONS.md         # 기획서 작성 규칙/템플릿
└── CURRENT_CONTEXT.md          # 현재 작업 상태 (자동 업데이트)
```

**결과**: AI가 프로젝트의 'DNA'를 학습하여, 이후 모든 기획 시 기존 스타일과 완벽한 일관성을 유지합니다.

## 5. 아이디어를 기획으로 변환하는 체계적인 워크플로우

### 4단계 기획 프로세스

| 단계 | 명령어 | 설명 |
|------|--------|------|
| 1 | `/ux plan` | 아이디어를 구체화하고 PRD 문서를 자동 생성 |
| 2 | `/ux flow` | 사용자 플로우 다이어그램과 IA를 설계하고 문서화 |
| 3 | `/ux spec` | 플로우 문서를 기반으로 상세 화면 기획서 생성 |
| 4 | `/ux tasks` | 기획서를 기반으로 구현 태스크를 분해하고 Worktree 생성 |

---

## 6. 모든 진행 상황의 시각화: Worktree

개발 진행 상황을 실시간으로 추적하고 시각화하는 CLI 기반 작업 트리입니다.

### 구조 예시

```
(Epic) SKuber 대시보드 리디자인 [Progress: 60%]
│
├─ (Story) 모니터링 화면 UX 개선 [Progress: 75%]
│   ├─ ✅ (Task) TASK-001: 사용자 리서치 및 페인포인트 분석
│   ├─ 🔄 (Task) TASK-002: 와이어프레임 작성
│   ├─ 🚫 (Task) TASK-003: UI 디자인 (블로커: 디자인 시스템 미확정)
│   └─ ○ (Task) TASK-004: 디자인 QA 및 핸드오프
```

### 상태 관리

| 상태 | 아이콘 | 설명 |
|------|--------|------|
| 완료 | ✅ | 작업 완료 |
| 진행중 | 🔄 | 현재 진행 중인 작업 |
| 블로커 | 🚫 | 차단된 작업 (사유 표시) |
| 대기 | ○ | 아직 시작하지 않은 작업 |

### 자동화

- 진행률이 자동으로 계산됨
- 명령어(`start`, `done`, `block`)로 상태를 쉽게 변경

---

## 7. 기획과 관리를 잇는 실시간 JIRA 양방향 동기화

### CLI에서 JIRA로 자동 연동

```bash
$ /worktree start TASK-002
$ /worktree done TASK-002
$ /worktree block TASK-003 "디자인 시스템 미확정"
```

### JIRA 보드 자동 업데이트

| To Do | In Progress | Done |
|-------|-------------|------|
| TASK-003: UI 디자인 [Blocked] | TASK-001: 사용자 리서치 | TASK-002: 와이어프레임 |
| TASK-004: 디자인 QA | | |

**가치**: PM과 관리자는 JIRA 대시보드에서 코드를 보지 않고도 팀의 실제 진행 상황을 실시간으로 파악할 수 있습니다.

## 8. 신경 쓰지 않아도 지켜지는 품질: 자동 활성화 스킬

기획자의 특정 키워드나 행동을 감지하여 내장 스킬이 자동으로 품질 규칙을 적용합니다.

| 스킬명 | 트리거 | 역할 |
|--------|--------|------|
| `ux-writing` | 기획서 작성 시 | UX 라이팅 가이드라인 준수 |
| `accessibility` | UI 설계 시 | WCAG 접근성 기준 체크 |
| `design-system` | 컴포넌트 관련 작업 시 | 디자인 시스템 규칙 참조 |
| `handoff-spec` | 핸드오프 문서 작성 시 | 개발팀 전달 스펙 포맷 적용 |
| `manual-template` | 매뉴얼 작성 시 | 사용자 매뉴얼 템플릿 적용 |

## 9. AI의 '건망증' 해결: 중단 없는 컨텍스트

### 자동 저장/로드

- 세션 시작 시 이전 상태를 자동으로 로드
- 종료 시 현재 상태를 저장

### Context Compact

- 대화가 길어져도 핵심 정보(규칙, 작업 상태)는 `CURRENT_CONTEXT.md`에 영구 보존

### 수동 복원

- `/ux restore-context` 명령어로 언제든지 핵심 컨텍스트를 다시 불러올 수 있음

## 10. Figma 연동: 디자인 토큰 동기화

### 연동 방식

Figma MCP 서버를 통해 디자인 시스템 정보를 자동으로 가져옵니다.

```bash
/ux figma-sync [Figma 파일 URL]
```

### 실행 결과

1. **디자인 토큰 추출** → `DESIGN_TOKENS.md` 업데이트
   - 컬러 팔레트 (Primary, Secondary, Semantic colors)
   - 타이포그래피 스케일
   - 스페이싱/그리드 시스템

2. **컴포넌트 목록 추출** → `UX_PATTERNS.md` 업데이트
   - Button variants
   - Input fields
   - Card components
   - Navigation patterns

3. **기획서 작성 시 자동 참조**
   - "버튼을 사용할 때" → `UX_PATTERNS.md`의 Button 규칙 자동 첨부
   - "색상 지정 시" → `DESIGN_TOKENS.md`의 컬러 팔레트 참조

## 11. 사용자 매뉴얼 제작 기능

### 매뉴얼 전용 명령어

| 명령어 | 용도 |
|--------|------|
| `/ux manual init [제품명]` | 매뉴얼 프로젝트 초기화, 기본 구조 생성 |
| `/ux manual outline` | 기능 요건서 기반 목차 자동 구성 |
| `/ux manual page [섹션명]` | 특정 기능의 매뉴얼 페이지 초안 작성 |
| `/ux manual build` | 전체 매뉴얼 PPT 파일 빌드 |

### 매뉴얼 문서 구조

```
.ux-docs/manuals/[제품명]/
├── MANUAL_OUTLINE.md       # 매뉴얼 목차 및 구조
├── MANUAL_STYLE_GUIDE.md   # 매뉴얼 스타일 가이드
├── sections/
│   ├── 00_cover.md         # 표지
│   ├── 01_overview.md      # 서비스 개요
│   ├── 02_getting_started.md
│   ├── 03_feature_xxx.md
│   └── ...
├── assets/
│   └── screenshots/        # 스크린샷 저장 위치
└── output/
    └── [제품명]_Manual_v1.0.pptx
```

### 매뉴얼 페이지 유형

| 유형 | 구성 |
|------|------|
| **기능 소개 페이지** | 좌측 기능 설명 + 우측 UI 스크린샷 + 하단 주요 포인트 |
| **단계별 가이드 페이지** | Step 1, 2, 3 순차 구성 + 각 단계별 스크린샷 |
| **표/비교 페이지** | 기능 비교표, 옵션 설명표 |

### 매뉴얼 제작 플로우

```
1. /ux manual init "SKuber v2.0"
   → manuals/SKuber_v2.0/ 폴더 구조 생성
   → MANUAL_STYLE_GUIDE.md 템플릿 생성

2. /ux manual outline
   → FUNCTIONAL_REQUIREMENTS.md 분석
   → 자동 목차 생성 (섹션별 .md 파일 생성)

3. /ux manual page "클러스터 모니터링"
   → sections/03_02_cluster_monitoring.md 생성
   → [스크린샷 필요] 표시로 이미지 삽입 위치 지정

4. /ux manual build
   → 모든 섹션 .md 파일 취합
   → PPT 템플릿에 맞춰 슬라이드 생성
   → output/SKuber_Manual_v2.0.pptx 출력
```

## 12. 사용 시나리오: 신규 기능 기획

### 전체 워크플로우

```
PM: JIRA에 '실시간 비용 알림' Epic 생성
    ↓
UX기획자: `/ux plan 실시간 비용 알림` → PRD 자동 생성
    ↓
UX기획자: `/ux flow` → 사용자 플로우 다이어그램 생성
    ↓
UX기획자: `/ux spec` → 상세 화면 기획서 생성
    ↓
UX기획자: `/ux tasks` → Worktree에 구현 태스크 자동 생성
    ↓
UX기획자: `/jira-push` → Worktree의 태스크들이 JIRA 이슈로 자동 생성
    ↓
UI디자이너: 디자인 작업 시작 (JIRA 상태 'In Progress'로 자동 변경)
    ↓
UX기획자: `/ux handoff` → 개발팀 전달용 스펙 문서 생성
    ↓
개발자: 작업 완료 → JIRA 상태 'Done'으로 자동 변경
```

**결과**: 체계적인 프로세스를 통해 일관된 품질과 투명한 진행 상황 공유를 모두 달성합니다.

- ✅ 일관된 기획 품질 및 문서 구조

## 15. 지금 바로 시작하세요

### 1단계: 설치 (1분)

```bash
git clone [repository_url]
cp -r .claude /path/to/your-project/
```

### 2단계: JIRA 연동 (선택 사항)

```bash
export JIRA_API_TOKEN='your-api-token'
/jira-init YOUR_PROJECT_KEY
```

### 3단계: Figma 연동 (선택 사항)

```bash
claude mcp add --transport http figma https://mcp.figma.com/mcp
```

### 4단계: 첫 명령어 실행

- **새 프로젝트**: `/ux init 'My new amazing feature'`
- **기존 프로젝트**: `/ux onboard`
